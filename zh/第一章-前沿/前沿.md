# 前言

## 欢迎来到 Nanobot 的世界

2025年底，OpenClaw（前身为 Clawdbot/Moltbot）以惊人的速度席卷了开发者社区——一周内斩获 10 万 GitHub Star，三个月突破 21 万。这个"能为你执行操作的个人 AI 助手"展示了 AI Agent 的无限可能：连接 WhatsApp、Telegram、Slack 等 12+ 平台，执行浏览器自动化、语音控制、定时任务，甚至拥有实时画布和配套移动应用。

然而，当我们满怀期待地打开 OpenClaw 的代码仓库时，43 万行 TypeScript 代码的庞大规模让许多人望而却步。复杂的模块耦合、晦涩的抽象层级、陡峭的学习曲线——**想要真正理解 AI Agent 的核心原理，似乎变成了一件遥不可及的事情**。

这就是 **Nanobot** 诞生的原因。

## 为什么要写这本阅读文档？

### Nanobot 的初心

2026年2月，香港大学数据科学实验室（HKUDS）推出了 Nanobot——一个仅用约 **4,000 行 Python 代码** 实现的超轻量级 AI Agent 框架。这不是对 OpenClaw 的简单模仿，而是一次对"AI Agent 究竟需要多少代码才能工作"的深度思考。

Nanobot 的核心洞察令人深思：

> 既然 AI Agent 的核心逻辑本质上就是一个循环——接收输入、思考判断、执行动作、输出响应——为什么不能用更清晰、更直观的方式来实现？

```python
while True:
    1. 接收输入（用户消息 + 上下文）
    2. 处理判断（LLM 分析：回复 or 调用工具）
    3. 执行动作（工具调用 → 获取结果 → 反馈）
    4. 输出响应（直接回复 or 继续循环）
```

这个朴素的设计哲学，让 Nanobot 成为了理解 AI Agent 原理的最佳入口。

### 编写本书的动机

作为 OpenClaw 生态的参与者，我深刻体会到：**理解 Nanobot，就是理解 OpenClaw 的最佳捷径**。

Nanobot 就像一本"AI Agent 教科书"——代码结构清晰、模块职责单一、函数调用关系直观。当你真正读懂了这 4,000 行代码，再回头去看 OpenClaw 的 43 万行代码，你会发现它们遵循着相同的设计模式，只是复杂度的叠加而已。

**本书的目标很简单**：
- 带你逐行读懂 Nanobot 的核心源码
- 帮助你建立对 AI Agent 架构的系统性认知
- 为你后续深入 OpenClaw 生态打下坚实基础

### 优势分析总结

**Nanobot 的独特优势**：

1. **极致轻量**：99% 的代码削减，让理解成本降至最低
2. **Python 生态**：纯 Python 实现，与数据科学生态无缝衔接
3. **亚洲平台优先**：原生支持飞书、钉钉、QQ 等国内常用平台
4. **研究友好**：清晰的代码结构，是学习和二次开发的理想选择
5. **MCP 原生支持**：率先支持 Model Context Protocol，工具扩展更灵活

**适用场景建议**：

```
你的需求是什么？
│
├─→ "我想搞懂 Agent 原理"
│       └─→ Nanobot（Python，代码像教科书）
│
├─→ "我要一个私人助理，安全可控"
│       └─→ NanoClaw（容器隔离，AI-native）
│
├─→ "我要在树莓派/嵌入式设备上跑"
│       └─→ ZeroClaw（<5MB 内存）或 PicoClaw（$10 硬件）
│
├─→ "我需要 MCP 协议支持"
│       └─→ Nanobot（原生 MCP）
│
└─→ "我要生产环境，功能最全"
        └─→ OpenClaw（20+渠道，生态完善）
```

## 本书的结构

为了帮助你系统地掌握 Nanobot，本书将按照以下结构组织：

1. **快速开始**：2 分钟部署你的第一个 Nanobot
2. **架构概览**：理解 Nanobot 的整体设计理念
3. **核心模块解析**：
   - Agent 循环（loop.py）—— 500 行代码的灵魂
   - 记忆系统（memory.py）—— 如何让 AI 记住对话
   - 工具系统（tools/）—— 扩展 Agent 的能力边界
   - 渠道接入（channels/）—— 连接各大聊天平台
   - 消息总线（bus/）—— 组件间的通信机制
4. **进阶主题**：
   - MCP 协议集成
   - 自定义 Skill 开发
   - 本地模型部署（vLLM）
5. **从 Nanobot 到 OpenClaw**：如何迁移和扩展
